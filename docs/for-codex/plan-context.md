# Plan Context

## Goal

採用代行業務のDXアプリ（歩留まり管理・架電履歴記録）のV1を初期構築する。
スプレッドシート+GAS運用からの脱却を図り、高速・セキュア・拡張性のあるモダンなWebアプリケーション（Next.js + Cloudflare Pages/D1）基盤を作る。

## Constraints

- 利用者は社内メンバー10名程度
- アプリへのアクセスにはGoogleアカウントによるログイン（OAuth）が必須
- データベースは無料で運用可能なCloudflare D1を利用
- CTI（電話システム連携）は今回はスコープ外
- 選考フェーズのカスタマイズ機能はスコープ外（書類・1次・2次/最終で固定）

## Draft plan

1. Next.js (App Router) アプリの雛形作成と Cloudflare Pages へのデプロイ準備
2. Cloudflare D1データベースのスキーマ設定とマイグレーション (Drizzle ORM)
3. Auth.js によるGoogle OAuth認証実装とミドルウェア保護
4. データベースのCRUD処理 (Server Actions / API Routes) 定義
5. フロントエンド実装1：応募者一覧・詳細画面（各種ステータスや日程の入力）
6. フロントエンド実装2：企業一覧画面（歩留まり集計表の表示、応募日・発生日フィルタ）
7. フロントエンド実装3：架電ログ機能（いつ、誰が、誰に、何回目、メモの記録）

## Open questions
