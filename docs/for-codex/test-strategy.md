# TDD Test Strategy (Phase 9)

本ドキュメントは、`@[/tdd]` ワークフローにおける Step 1: テスト戦略設計（Gate 1）のためのテスト観点・ロジック要件をまとめたものです。

## 1. 応募者一覧：行クリックでの詳細画面遷移

- **正常系**
  - `<tr className="cursor-pointer">` な行の任意の位置をクリックした際、`router.push('/applicants/[id]')` によって正しく詳細画面へ遷移すること。
- **異常系・エッジケース**
  - 行の中に含まれる「詳細を見る」などの既存の `<a>` タグをクリックした際、イベントのバブリングによる2重遷移エラーや予期せぬ挙動が起こらないこと（`e.stopPropagation` の考慮）。

## 2. 架電ログからの新規架電登録

- **正常系**
  - 架電ログ画面の中に「新規架電登録フォーム」が存在し、応募者をプルダウン（または検索コンボボックス）で選択して、通電フラグ・メモとともに保存できること。
- **異常系・エッジケース**
  - 応募者が1人も存在しない場合、プルダウンが空になり正常に無効化（disabled）されること。
  - 応募者の選択なしでサブミットされた場合、サーバーアクション側またはクライアントバリデーションで弾かれること。

## 3. 歩留まり管理（企業別）の追加指標計算

- **計算ロジック（仕様の仮決め・要確認）**
  - **有効応募からの面接設定率**:
    - 分子: `primaryScheduled`, `secScheduled`, `finalScheduled` のいずれかが true である応募者数
    - 分母: `isValidApplicant` が true である応募者数
  - **有効応募からの着席率**:
    - 分子: `primaryConducted`, `secConducted`, `finalConducted` のいずれかが true である応募者数
    - 分母: `isValidApplicant`
  - **有効応募からの内定率**:
    - 分子: `offered`
    - 分母: `isValidApplicant`
  - **有効応募からの入社率**:
    - 分子: `joined`
    - 分母: `isValidApplicant`
  - **面接前辞退率**:
    - 分子: `interviewDeclinedBefore` または `secDeclinedBefore` が true の応募者数
    - 分母: 「面接設定ができた人数（上記、面接設定率の分子）」
  - **内定後辞退率**:
    - 分子: `offerDeclined`
    - 分母: `offered`
- **異常系・境界値**
  - 分母が `0`（有効応募が0件など）になる場合、`ZeroDivisionError` や `NaN` や `Infinity` にならず、「0%」または「-」として安全にフォールバック表示すること。

## 4. サイドバーの折り畳み昨日

- **正常系**
  - サイドバー最下部などに「折り畳みトグルボタン」を配置。
  - 折り畳み状態（Collapsed）では、テキストラベルが消えアイコンだけが表示されるコンパクトな幅（例: `w-16` 等）になること。
- **異常系・境界値**
  - 状態変更アニメーション中にメインコンテンツ領域（表など）がガタつかず、正しく追従・リサイズされること。
  - レスポンシブ環境（モバイル等）での表示に悪影響を与えないこと。

## 5. UIモダナイゼーション（機能影響ゼロのUI改修）

- **正常系（ビジュアル・UXの確認）**
  - **ログイン画面**: 余白・シャドウが洗練され、エラー表示が「文字＋色＋アイコン」で正しく表示されること。Tabキーでのフォーカス移動が適切であること。
  - **一覧画面**: テーブルの行ホバー時や選択時に、視覚的なフィードバック（背景色変更など）が明確に機能すること。ヘッダーやフィルタが視認性高く整理されていること。
  - **フォーム**: Input, Select, Checkboxなどが統一されたフォーカスリングを持ち、ラベル・エラーのコンポーネント配置が明確化されていること。
  - **共通・レスポンシブ**: デスクトップ・モバイルの両方でレイアウト崩れが発生しないこと。要素の高コントラストと可読性が保たれていること。
- **異常系（回帰テスト・機能デグレードの防止）**
  - 本改修によって、既存の**CRUD操作、バリデーション、APIリクエスト、DB保存、認証、ルーティング機能が一切破壊されていないこと**（UIレイヤーの className や構造調整のみに留めていることの担保）。
  - ボタンやリンクのクリック領域が他要素によって阻害されず、既存のクリックイベントが正常に発火すること。
